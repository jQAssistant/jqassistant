== 01 - Detailed Logging Implementation (SLF4J)

=== Purpose
This document provides concrete code examples and technical constraints for implementing the logging decision defined in the parent module.

=== Implementation Patterns

==== Logger Initialization
Every class should initialize the logger as follows to ensure the class name is correctly captured:

[source,java]
----
private static final Logger log = LoggerFactory.getLogger(MyClass.class);
----

==== Usage of SLF4J Fluent API
Where possible, use the fluent API for better readability:

[source,java]
----
log.atInfo()
   .addKeyValue("userId", user.getId())
   .log("User successfully logged in");
----

=== Log Message Requirements

* **Correlation ID**: Every log entry triggered by a web request must include the `X-Correlation-ID` from the MDC (Mapped Diagnostic Context).
* **No Sensitive Data**: Strings must be scrubbed of passwords, IBANs, or private addresses before passing to the logger.
* **Exceptions**: When logging an error, the exception must be the final parameter:
** `log.error("Action failed for id {}", id, exception);`

=== Configuration
The `logback-spring.xml` or `log4j2.xml` is configured to output JSON format in production environments to support ELK/Splunk integration.

=== Examples

[source,java]
----
public void processOrder(Order order) {
    log.debug("Starting process for order: {}", order.getId());
    try {
        // business logic
    } catch (Exception e) {
        log.error("Critical failure during order processing", e);
        throw new BusinessException(ErrorCode.PROCESSING_ERROR);
    }
}
----
